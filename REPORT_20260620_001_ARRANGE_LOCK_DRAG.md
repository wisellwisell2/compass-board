## 作業概要
人生コンパスで「画面サイズ変更・ピンチ俯瞰の状態でカードを触ると、触った瞬間に別の場所へ飛ぶ」不具合を、`arrange.html` を直して解消した。あわせて、要望どおり **ロック/解除トグル**（既定ロック＝安全）を付け、現在の状態を**色＋アイコン＋ボタン下の小さな文字**で曖昧さなく示すようにした。

カードデータの正本はブラウザの `localStorage["compassBoard.v1"]` であり、本作業ではデータを書き換えず、それを読み込んで配置を編集し同じキーへ保存する設計とした。

## 実装内容
`arrange.html` は従来どおり `index.html` を iframe で読み込み、iframe 内へ補正スクリプトを注入する方式。注入スクリプトは `index.html` がクラシックスクリプト（`type=module` でない）であるため、内部の `state` / `metrics` / `renderBoard` / `touchCard` / `save` / `handleCardTap` 等を名前で参照できる。

1. **飛びの根絶（座標補正＋native無効化）**
   - `.card` への `pointerdown` を `document` の capture フェーズで奪取し、`stopImmediatePropagation()` で `index.html` 純正のドラッグ処理（[index.html:859](index.html:859)、zoom 無補正で飛ぶ版）を**完全に無効化**。これによりカードは注入側のロジックでしか動かない。
   - ドラッグ座標は毎イベントで `boardScale() = board.getBoundingClientRect().width / board.offsetWidth`（≈現在の CSS `zoom`）を計算し、画面座標をボード座標へ `(/scale)` 変換。`zoom` 変更・画面リサイズに毎回追従する。
   - ドロップ時はカード中心 X からフェーズを判定し、`card.phaseId` / `card.x = gx - phase.off` / `card.y = gy` を保存（`index.html` 純正と同じ規約）。`touchCard()`→`save()`→`renderBoard()`。

2. **ロック/解除トグル＋状態表示**
   - 既定は **ロック**（カード固定・絶対に動かない・安全）。
   - 状態は3要素で明示：色（ロック＝青グレー／移動＝緑）＋アイコン（🔒／🖐）＋語（「ロック中」／「移動モード」）。さらにボタン直下のキャプションで「今の挙動」と「もう一方への切替方法」を併記。
     - ロック中: 「カードは固定 ／ タップで詳細。動かすにはロックを解除」
     - 移動モード: 「ドラッグで移動・指を離すと自動保存 ／ タップで詳細。固定するにはロック」
   - `保存` ボタン（明示保存。押下時「保存しました ✅」を一時表示）／`通常画面` リンクを併設。ドロップ毎に自動保存もする。

3. **タップ／ピンチの保護**
   - ロック中・移動モードのどちらでも、移動しきい値（9px）未満で離した場合は `handleCardTap()` を呼び、純正の「タップで選択→詳細／サブカード選択」を再現（詳細閲覧を維持）。
   - 2本目のポインタが下りたら進行中のカード操作を中断し、空き領域でのピンチ俯瞰に委ねる。

4. **データ安全**
   - 保存は `save()`（失敗時のみ `localStorage.setItem('compassBoard.v1', …)`）。`localStorage.clear()` や空での上書きは一切しない。座標 `x`/`y` と `phaseId` のみ更新（`category` 等の既存フィールドは不変）。
