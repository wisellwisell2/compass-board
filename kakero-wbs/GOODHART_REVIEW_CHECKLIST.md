# Goodhart Review Checklist

作成日: 2026-06-16

## 目的

ゲーム要素を追加するときに、KAKEROの人生マップが嘘にならないか確認する。

ゲームを遊んでいたら自然に人生マップが進むのはよい。  
しかし、ゲーム報酬のためにユーザーが人生マップをハックし始めたら、KAKEROの中核価値が壊れる。  
したがって、ゲーム報酬と人生マップ指標を機械的に連動させない。

## チェックリスト

| ID | 観点 | 確認すること | 判定 |
| --- | --- | --- | --- |
| G-01 | 進捗盛り | ゲーム報酬のためにユーザーが進捗を盛る設計になっていないか | OK / NG / 要修正 |
| G-02 | けんこうポイント盛り | けんこうポイントを高く見せるほど得する設計になっていないか | OK / NG / 要修正 |
| G-03 | カード乱造 | カードを雑に増やすほど報酬が増える設計になっていないか | OK / NG / 要修正 |
| G-04 | 正直さ | 人生マップの正直さを壊していないか | OK / NG / 要修正 |
| G-05 | 自然進行 | ゲームを遊んでいたら自然にマップが進む設計か | OK / NG / 要修正 |
| G-06 | ハック誘導 | ゲームのためにマップをハックする設計になっていないか | OK / NG / 要修正 |

## NG例

- カードを追加するだけで称号や報酬が増える
- けんこうポイントを高くするほどレア報酬が出る
- 完了数を増やすほど報酬が出るため、ユーザーが雑な小タスクを量産する
- 「正直な赤信号」より「嘘の青信号」の方が得をする

## OK例

- 正直に状態を入力した結果、今の状態に合う励ましや見た目が出る
- タスク完了数ではなく、盤面を見返す、直す、整理する行為が自然に起きる
- 報酬は人生マップ指標に直結せず、利用の節目や発見として出る
- 悪い状態を入力しても損をしない
