# KAKERO WBS

作成日: 2026-06-15

## 1. 前提

このWBSは、KAKEROの **PoC -> 商品 walking skeleton -> 技術GO / 体験GO / 事業GO 判定** までを対象にする。

WBS内で事業判断・価格・仕様未確定事項は勝手に確定しない。判断が必要なものは `OPEN_QUESTIONS.md` に分類し、作業停止の理由にしない。

## 2. 作業分類

| 分類 | 扱い |
| --- | --- |
| 決定済み | 作業化してよい。WBSの本線に入れる |
| PoC必要 | 実装前または本線統合前に成立確認する |
| 未確定 | 判断待ち。`OPEN_QUESTIONS.md` に記録し、仮置きで進める |
| 後回し | 商品 walking skeleton には入れない |

## 3. 決定済み

| 項目 | 決定内容 |
| --- | --- |
| 初期形態 | Web/PWA + Cloudflare 前提 |
| 保存方式 | 初期保存方式は作りやすさ優先。ただし将来的には端末ローカル保存/移行を有力候補とする |
| AIカード化 | Haiku API 中継でカード化 |
| 会話ログ | 会話ログとしては保持しない |
| 記憶 | 「覚えてほしいことリスト」を盤面データとして持つ |
| データ構造 | OKF風の自前JSON。正式OKF準拠は不要 |
| 人生コンパス | 既存 compass-board はソースあり前提で取り込む |
| walking skeleton | 単なる技術検証ではなく、最低限遊べる商品状態 |
| 初回体験 | チュートリアルで10個前後のカード登録体験まで含める |
| 課金導線 | 本番決済ではなく、プラン/価格ページとして扱う |
| 遊びレイヤー | 入口と継続エンジンは遊び。本質的payoffは人生が盤面になること |
| 遊びの強度 | 遊びは飾りではなく、ドパガキがしがみつくレベルの継続エンジンとして検証する |
| Goodhart回避 | ゲーム報酬と人生マップ指標を機械的に連動させない |
| 価格原則 | 基本は0円と3000円のバーベル。中途半端な価格階段を初期に置かない |

## 4. 全体規模感

副業ペースは 1日2-4時間、週4-5日稼働を想定する。1人日をフルタイム1日相当として見積もるため、副業カレンダーではおおむね **1人日 = 2-3暦日** として読む。

| 範囲 | 人日目安 | 副業ペース目安 | ゴール |
| --- | ---: | --- | --- |
| Phase 0-2 | 9-14人日 | 3-5週間 | データ骨格とHaikuカード化PoC |
| Phase 3-4 | 15-24人日 | 5-8週間 | コンパス/けんこうマップ反映PoC |
| Phase 5 | 22-38人日 | 7-12週間 | 商品 walking skeleton、遊びレイヤーPoC、原則確認ページ |
| Phase 6 | 5-8人日 | 1-2週間 | 技術GO / 体験GO / 事業GO 判定 |
| 合計 | 52-84人日 | 4.5-7か月目安 | 最低限遊べる商品状態とGO判定 |

## 6. 後回し

| 項目 | 理由 | 再検討タイミング |
| --- | --- | --- |
| 6体召喚体の作り込み | walking skeletonの成立に不要 | 体験GO後 |
| 超丁々発止の本実装 | 原価・演出・価値設計が未確定 | 事業GO前後 |
| Stripe等の本番決済 | 初期はプラン/価格ページで足りる | 有料検証開始時 |
| 複数端末同期 | MVP外 | 保存方式の本決定後 |
| ローカルLLM本線化 | 初期本線はHaiku API | 原価/プライバシー再評価時 |
| 正式OKF準拠 | 過剰設計 | 外部連携が必要になった時 |
| ミニゲーム | 本線価値検証に不要 | Retention課題が見えた後 |
| 広告運用 | 最初は手売り/少人数検証が向く | 入口/体験の勝ち筋確認後 |
| 本格的な運用/問い合わせ体制 | 商品WS段階では重い | 公開範囲拡大前 |
| 卵の形への固定 | 突発イベント/隠し報酬の代表案にすぎない | 予期しない報酬PoC後 |
| ゲーム報酬と人生マップ指標の機械連動 | Goodhart化で中核価値が壊れる | 原則として避ける |

## 7. テスト観点

| 観点 | 合格条件 |
| --- | --- |
| Haiku出力 | JSONが壊れず、願望が6フィールドまたは内部スキーマへ落ちる |
| 脱線拒否 | 雑談・天気・愚痴・AIへの話しかけを拒否できる |
| 初回体験 | 5-10カード登録まで到達できる |
| 保存/復元 | 盤面と編集結果が残る |
| 手編集 | AI出力のズレをユーザーが直せる |
| けんこうマップ | 単なるToDoではなく、生活状態の変化として見える |
| 価格導線 | プラン/価格ページで有料エンジンの価値が説明できる |
| 遊びレイヤー | かわいい、楽しい、変な称号、育て要素、突発イベント、隠し要素が毎日開く理由になる |
| Goodhart回避 | ゲーム報酬のために人生マップをハックしたくなる構造になっていない |

## 8. GO判定

| 判定 | 見るもの | 合格の目安 |
| --- | --- | --- |
| 技術GO | 作れるか | Haiku、盤面、編集、保存、けんこう反映が一通り通る |
| 体験GO | 分かるか・続くか | 10カード前後で「自分の人生が盤面になった」と感じ、遊びレイヤーが継続理由になる |
| 事業GO | 売れるか | ICP、課題、価格、継続、集客、粗利の仮説と検証材料がある |
